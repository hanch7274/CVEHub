CVEHub 개발 진행 상황

CVEHub 프로젝트의 현재 개발 완료 수준과 향후 계획을 시각적으로 나타냅니다.

전체 진행률

60%

현재 프로젝트의 약 60%가 완료되었습니다.

현재까지 구현된 기능

  • CVE 관리: 목록 조회, 상세 보기, 생성, 수정, 삭제 등 핵심 CRUD 기능
  • 데이터 자동 수집: Nuclei, Metasploit, EmergingThreats 기반 PoC, Snort Rule 정보 자동 수집 및 연동
  • 협업 및 정보 공유: CVE별 댓글(멘션 포함), 참조 링크, PoC 정보 공유 기능
  • 실시간 동기화: WebSocket(Socket.IO)을 통한 데이터 변경 실시간 전파 (CVE 업데이트, 댓글 등)
  • 사용자 관리 및 인증: JWT 기반 인증, 역할 기반 접근 제어, 사용자 정보 관리
  • 활동 이력 추적: 사용자 및 시스템 활동 로그 기록 및 조회 기능
  • 알림: 사용자 멘션, 주요 업데이트 등에 대한 실시간 알림 기능
  • 캐시 관리: Redis 및 React Query 캐시 상태 시각화 및 관리 기능 (관리자용)

향후 개발 예정

  • 크롤러 고도화: 보안 뉴스, 국내/외 벤더사 보안 블로그 등 추가 정보 소스 자동 수집
  • 뉴스/포스팅 게시판: 수집된 정보 게시 및 CVE 매칭을 통한 실시간 분석 상태 연동
  • Local LLM 도입:
    • NVD 설명 등 외부 정보 자동 번역
    • PoC 기반 Snort Rule 자동 생성 제안
  • PCAP 관리 기능:
    • PCAP 파일 업로드 및 관리
    • 저장된 Snort Rule과 PCAP 웹 기반 테스트 기능
  • 통계 및 연동 강화:
    • 주간 CVE 업데이트 통계 리포트 제공
    • 관제 시스템 연동을 통한 탐지 내역 조회 및 분석
